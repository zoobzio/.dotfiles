# Recon

Survey the ecosystem before the briefing. Understand what packages are available, what gaps exist, and whether the plan is executable with the parts we have.

## Execution

1. Read your memories from prior issues — packages that were missing, logistics problems, board structures that failed
2. Survey the ecosystem using `/grok` — what packages exist for the domains the issue will touch
3. Check whether dependencies the crew relied on last time are still current
4. Identify gaps — capabilities the plan will need that no package currently provides
5. Produce a briefing contribution

## What To Survey

| Area | What to check |
|------|---------------|
| Package availability | Do we have packages for what the plan requires? |
| Ecosystem gaps | Any domain where the crew will need something we do not have? |
| Dependency state | Are packages the crew used before still maintained and current? |
| Execution feasibility | Can the plan be broken into tasks that unblock in a buildable order? |
| Prior logistics | Memories from previous issues — what slowed us down, what was missing |

## Output

Briefing contribution covering:

- **Available packages** — what we have that the plan can use
- **Gaps** — what we need and do not have
- **Logistics concerns** — anything that affects whether the plan is executable
- **Recommendations** — packages to evaluate, order-of-operations concerns
