# Repo

Managing the repository — issues, health, PRs, and everything that keeps the package honest. Read the relevant sub-file for the work at hand.

| File | When |
|------|------|
| `recon.md` | Before the briefing — survey repo state, issues, health |
| `health.md` | Before opening a PR and periodically — lint, security, issues, dependencies |
| `open.md` | Committing the work and opening the pull request |
| `monitor.md` | Watching CI and detecting failures |
| `triage.md` | Reading reviewer comments and deciding what to do about them |

Do not load all sub-files upfront. Read the one that matches what you are doing right now.
