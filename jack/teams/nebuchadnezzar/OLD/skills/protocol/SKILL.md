# Protocol

The situational playbook for how the Nebuchadnezzar crew handles specific circumstances. These protocols supplement the rules in ORDERS.md.

Read the specific protocol file when a situation calls for it. Do not load all protocols upfront.

| Protocol | File | When to Read |
|----------|------|--------------|
| Briefing | `briefing.md` | Before the briefing — recon, contributions, structure |
| Task Board | `board.md` | Before interacting with the task board during Build |
| Communication | `communication.md` | When deciding whether to message or update the board |
| Bug | `bug.md` | When a defect is discovered during testing |
| Escalation | `escalation.md` | When a problem exceeds your domain |
| Support | `support.md` | When you need Morpheus, or when Morpheus arrives to help |
| Coordination | `coordination.md` | For phase handoffs, builder-tester flow, and regression transitions |
| NEBUCHADNEZZAR | `nebuchadnezzar.md` | Before posting any external communication |
