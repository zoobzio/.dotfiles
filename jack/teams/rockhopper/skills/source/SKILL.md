# Source

The implementation playbook. Read the specific sub-file when a concern is relevant to the current task. Do not load all sub-files upfront.

| Concern | File | When to Read |
|---------|------|--------------|
| Recon | `recon.md` | Before writing any code — survey what exists, how it's structured, what patterns govern it |
| Structure | `structure.md` | When creating files, packages, or organising code within a package |
| Generics | `generics.md` | When defining types, functions, or interfaces that operate on parameterised data |
| Errors | `errors.md` | When defining error types, sentinel errors, or handling error paths |
| Options | `options.md` | When writing constructors, configuration, or the functional options pattern |
| Concerns | `concerns.md` | When designing interfaces, composing types, or separating responsibilities |
| Boundaries | `boundaries.md` | When transforming data at package edges — parsing, validation, normalisation |
| Workspaces | `workspaces.md` | When working with multi-module repos, testing submodules, or provider submodules |
