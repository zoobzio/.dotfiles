# Build

The run. Research the problem, plan the approach, implement the solution. Every issue follows this shape.

Read the sub-files in order. Do not skip phases.

| File | Phase | When |
|------|-------|------|
| research.md | Research | First — understand the problem before planning |
| plan.md | Plan | Second — design the approach before implementing |
| implement.md | Implement | Third — write the code, test it, ship the PR |
