# Manage

The captain's playbook. Read the specific sub-file when a situation calls for it. Do not load all sub-files upfront.

| Concern | File | When to Read |
|---------|------|--------------|
| Recon | `recon.md` | Before the briefing — survey GitHub issues for related or blocking work |
| Briefing | `briefing.md` | When opening the briefing — structure, time-boxing, user updates |
| RFC | `rfc.md` | When handling a scope RFC — from agents or from the user after a veto |
| Issues | `issues.md` | When commenting on or augmenting GitHub issues |
| Labels | `labels.md` | When applying or updating labels on GitHub issues |
| PR | `pr.md` | When creating, commenting on, or triaging pull request feedback |
