# Protocol

The situational playbook for how the Rockhopper crew handles specific circumstances. These protocols supplement the rules in ORDERS.md.

Read the specific protocol file when a situation calls for it. Do not load all protocols upfront.

| Protocol | File | When to Read |
|----------|------|--------------|
| Briefing | `briefing.md` | At the start of every work item — the round-robin discussion protocol |
| Review | `review.md` | At the start of Review phase — independent review then round-robin |
| Task Board | `board.md` | Before interacting with the task board during Build |
| Communication | `communication.md` | When deciding whether to message or update the board |
| Support | `support.md` | When Fidgel is shifting to active builder or tester |
| Bug | `bug.md` | When a bug is discovered during testing |
| Escalation | `escalation.md` | When escalating to Fidgel (diagnostic) or Zidgel (scope RFC) |
| Coordination | `coordination.md` | For handoffs outside Build, or rewrite coordination |
| ROCKHOPPER | `rockhopper.md` | Before posting any external communication |
| Issue Labels | `labels.md` | When updating labels on GitHub issues |
