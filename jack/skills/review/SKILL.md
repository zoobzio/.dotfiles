# Review

Dual review playbook for the Review phase. Read the specific sub-file when the situation calls for it. Do not load all sub-files upfront.

| Domain | File | When to Read |
|--------|------|--------------|
| Logical | `logical.md` | When reviewing for requirements satisfaction, scope adherence, and user value |
| Technical | `technical.md` | When reviewing for spec correspondence, pattern correctness, and architecture soundness |
| Mechanical | `mechanical.md` | When running independent verification — build, test suite, documentation accuracy |
