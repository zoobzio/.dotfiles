# Docs

The documentation playbook. Read the specific sub-file when a concern is relevant to the current task. Do not load all sub-files upfront.

| Concern | File | When to Read |
|---------|------|--------------|
| Recon | `recon.md` | Before writing any docs — survey what exists, what's stale, what's missing |
| README | `readme.md` | When creating or updating a package README |
| Learn | `learn.md` | When writing overview, quickstart, concepts, or architecture docs |
| Guides | `guides.md` | When writing how-to guides — configuration, testing, best practices |
| Reference | `reference.md` | When writing lookup docs — cheatsheets, API tables, type catalogs |
| Cookbook | `cookbook.md` | When writing real-world recipes and complete runnable examples |
| Frontmatter | `frontmatter.md` | When structuring docs for the site — numbering, cross-references, metadata |
