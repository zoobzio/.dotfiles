# Recon Scope

Survey the issue landscape before the briefing. Understand what exists, what's been tried, and what context the crew should have.

## Execution

1. Read your memories from prior issues — patterns, scope decisions, recurring gaps
2. Read the issue or user request
3. Search for related issues — overlapping work, prior attempts, relevant context
4. Survey the issue labels and milestone state
5. Produce a briefing contribution

## What To Survey

| Area | What to check |
|------|---------------|
| The issue | Problem statement, context, existing criteria, reporter intent |
| Related issues | Overlapping scope, prior attempts at the same problem, linked issues |
| Labels and milestones | Phase state, priority signals, escalation history |
| Prior scope decisions | Memories from previous issues that constrained or expanded scope |

## Output

Briefing contribution covering:

- **The mission** — what the system owes and where the gap is
- **Related context** — issues, prior work, or decisions that affect this scope
- **Known risks** — scope concerns surfaced from memory or the issue landscape
- **Open questions** — things the crew needs to resolve before Plan
